> 刚开始想用python来测试多进程和多线程的优缺点的,写完代码发现不对，才想起python的全局解释器锁(GIL)的问题。所以只能测试一下多进程的速度了。  

用法: `./run.sh`

结果:  
4进程最快,第二是40进程,第三是8进程。  
因为我的CPU是4核的，所以4进程应该是最快的。但是第二竟然是40进程。我的理解是进程越多,需要不断的切换进程，会消耗时间，而8进程切换的次数少,应该更快。
